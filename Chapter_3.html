These problems use the samples from the posterior distribution for the globe tossing example. This code will give you a specific set of samples, so that you can check your answers exactly

length.out <- 1001
n_water <- 6
n_trials  <- 9

d <-
  tibble(p_grid     = seq(from = 0, to = 1, length.out = length.out),
         prior      = 1) %>% 
  mutate(likelihood = dbinom(n_water, size = n_trials, prob = p_grid)) %>% 
  mutate(posterior  = (likelihood * prior) / sum(likelihood * prior))

# How many samples would you like?
n_samples <- 1e4

# Make it reproducible
set.seed(3)

# Draw samples
samples <-
  d %>% 
  sample_n(size = n_samples, weight = posterior, replace = T)

3E1. How much posterior probability lies below p = 0.2?

SOLUTION

solution <- 
  samples %>% 
  filter(p_grid < .2) %>% 
  summarise(sum = n() / n_samples)

print(paste0("The posterior probability that lies below p=0.2 is ", solution$sum))
## [1] "The posterior probability that lies below p=0.2 is 6e-04"

But Why should we use only ten possible parameter values if our grid space has a thousand steps? Lets repeat the last step but using a thousan possible values, that is, approximating better the integral form of the Posterior predictive distribution: \[p(\hat{x}|data) = \int_{\Theta} p(\hat{x}|\Theta,data) p(\Theta|data) d\Theta\]

Whrere \(\hat{x}\) is the sampled data, \(\Theta\) the estimated parameter (percentage of water in our case) and \(p(\Theta|data)\) the posterior distribution.
